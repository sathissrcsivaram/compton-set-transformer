Transformermodel Architecture Workflow

Single-page workflow summary for the Compton source-localization heatmap model in train_compton_set_transformer.py

Stage Input Output Shape Key Details
1. Dataset Source /content/mydrive/MyDrive/Transformermodel/eventset.csv Raw CSV rows Loaded with pd.read_csv. Required columns include source, scatter, absorb, theta, and energy fields.
2. Feature Prep + Grouping Raw event rows 6 features per row
grouped by source		 Uses [Scatter_X, Scatter_Y, Absorb_X, Absorb_Y, Theta, Energy], converts theta to radians, and builds source_id from source coordinates.
3. Train / Val / Test Split Grouped sources 70% / 15% / 15% Prevents leakage across the same source. Grid statistics are computed from train only.
4. Tensor Builder Per-source events events: [B, 3000, 6]
mask: [B, 3000]
target: [B, 2500]		 Each source is padded or truncated to MAX_EVENTS = 3000. Target is a flattened 50 x 50 heatmap.
5. Model Inputs events, mask [B, 3000, 6], [B, 3000] Keras model takes event sequences and a boolean mask indicating valid events per source.
6. Shared Normalization events [B, 3000, 6] KerasNormalization(axis=-1) is adapted on the full training tensor before fitting.
7. Event Encoder Block 1 [B, 3000, 6] [B, 3000, 256] Dense -> GELU -> Dropout -> Dense -> LayerNorm with residual projection to 256 channels.
8. Event Encoder Block 2 [B, 3000, 256] [B, 3000, 256] Second residual event block with the same width. Keeps event-wise processing shared across time.
9. Event Encoder Block 3 [B, 3000, 256] [B, 3000, 128] Third residual block reduces event embedding width to 128.
10. ISAB Attention Layer [B, 3000, 128] + mask [B, 3000, 128] Induced Set Attention Block with 4 heads, 128 inducing points, dropout 0.10, and learnable inducing tokens.
11. Attention Pooling [B, 3000, 128] [B, 128] Masked attention scores over events. Produces a permutation-invariant pooled source representation.
12. Count + Prediction Head Pooled source vector + mask count [B, 128] Normalized event count is concatenated, then processed by Dense 128 -> Dropout -> Dense 128 -> LayerNorm.
13. Heatmap Output [B, 128] [B, 2500] Softmax over HEATMAP_H * HEATMAP_W = 50 * 50. Represents a probability distribution over source location cells.
14. Training Setup Train/val datasets Epoch loop Adam optimizer, KL-divergence style loss, batch size 16, max 300 epochs, ModelCheckpoint, ReduceLROnPlateau, EarlyStopping(patience=7).
15. Inference + Uncertainty Best saved weights Mean heatmap predictions Uses MC_DROPOUT_PASSES = 10 for stochastic forward passes during evaluation.
16. Final Outputs Predicted heatmaps Metrics + artifacts Saves best.weights.h5, final.weights.h5, training curves, histograms, side-by-side heatmaps, and predictions_test_map.csv.
Main configuration used: EVENT_HIDDEN=[256,256,128], MAX_EVENTS=3000, NUM_SAB_HEADS=4, ISAB_NUM_INDUCING=128, HEATMAP=50x50, BATCH_SIZE=16.

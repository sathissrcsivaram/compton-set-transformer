Transformermodel Change Report

Prepared on March 31, 2026

1. Project Background

The local project folder was located at /home/siv/Transformermodel. Initially, this folder was not a Git repository. The code existed locally, while the data files were also stored in the same folder.

2. What Was There Before

  • The folder contained the code file train_compton_set_transformer.py.
  • The folder contained the dataset file eventset.csv.
  • The folder contained a generate_events subfolder with helper code.
  • The folder also contained a large data/ directory.
  • The folder was not connected to GitHub as its own repository.

3. Original File Path Behavior

Earlier versions of the code used a local or Colab-root CSV path. One of the active forms used was:

CSV_PATH = "/content/eventset.csv"

That path assumes that eventset.csv is copied directly into the Colab runtime under /content.

4. Requested Storage Split

The requested final setup was:

  • Keep the code in GitHub.
  • Keep eventset.csv in Google Drive.
  • Run the code in Colab with Drive mounted.

5. Changes Made

Area Before After
Repository Local folder only, no Git repository for Transformermodel. A dedicated GitHub repository was created: https://github.com/sathissrcsivaram/Transformermodel.
CSV path /content/eventset.csv or prior Colab path assumptions. /content/mydrive/MyDrive/Transformermodel/eventset.csv to match the chosen Drive mount point.
Large files Local folder contained code and dataset together. Large local data files were excluded from GitHub so the repo only contains code.
Keras compatibility Custom layer used an older add_weight calling style. The layer was updated to the current Keras-compatible form using name="inducing".

6. Current Important Code Changes

6.1 CSV Path Update

The active code now uses:

CSV_PATH = "/content/mydrive/MyDrive/Transformermodel/eventset.csv"

This matches the Colab mounting pattern:

from google.colab import drive drive.mount('/content/mydrive')

6.2 Keras Error Fix

The failing code path previously produced this error:

TypeError: Layer.add_weight() got multiple values for argument 'shape'

The reason was that newer Keras versions interpret the first positional argument of add_weight differently. The code was updated from the older style to an explicit keyword-based form.

Corrected code:

self.I = self.add_weight( name="inducing", shape=(1, self.m, self.d), initializer="glorot_uniform", trainable=True )

8. Files Excluded From GitHub

To keep the repository lightweight and avoid uploading large local data, the following were excluded:

eventset.csv data/ __pycache__/ *.pyc

This means the dataset stays in Google Drive, while GitHub stores only the code and project structure.

9. Final Working Setup

The intended runtime setup is now:

  • Clone the code from GitHub into Colab.
  • Mount Google Drive at /content/mydrive.
  • Read the dataset from /content/mydrive/MyDrive/Transformermodel/eventset.csv.
  • Run train_compton_set_transformer.py from the cloned repository.

10. Conclusion

Before these changes, the project existed only as a local folder and used a CSV path that did not match the final Google Drive setup. After the changes, the code is stored in a dedicated GitHub repository, the large dataset is intentionally kept out of GitHub, the CSV path matches the chosen Colab mount point, and the Keras compatibility issue in the custom ISAB layer has been fixed.

Transformermodel Model Explanation Report

Detailed professor-facing explanation of the architecture, loss, activations, metrics, training strategy, and design choices used in train_compton_set_transformer.py.

1. What This Model Is Solving

This model estimates the source location of a Compton event set. Instead of predicting a single coordinate directly, it predicts a probability distribution over a 2D grid. In the code, the final output is a flattened 50 x 50 = 2500 cell heatmap. Each source is therefore represented by a spatial probability map.

The model input is not one event but a set of events belonging to one source. This is important because event order is not inherently meaningful. For that reason, the design is closer to a DeepSets / Set Transformer style model than to a simple sequence model.

2. End-to-End Workflow

  1. The CSV is loaded from Google Drive.
  2. Relevant columns are checked and Theta is converted to radians.
  3. Rows are grouped by source using a combined source_id.
  4. Sources are split into train, validation, and test sets.
  5. A 2D prediction grid is built from the training set only.
  6. Each source is converted into padded event tensors, masks, and a target Gaussian heatmap.
  7. The set model processes the event set and outputs a 2500-dimensional heatmap.
  8. The model is trained with a distribution-matching loss.
  9. The best weights are restored and evaluated with MAE and R² on source coordinates.

3. Input Data and Why These Features Are Used

The model uses these six event-level input features:

FEATURE_COLS = ["Scatter_X", "Scatter_Y", "Absorb_X", "Absorb_Y", "Theta", "Energy"]

Why these were chosen:

  • Scatter_X and Scatter_Y encode where the scattering interaction happened.
  • Absorb_X and Absorb_Y encode where absorption happened.
  • Theta contains angular information related to the Compton scattering geometry.
  • Energy carries physical information that helps disambiguate event patterns.

These features jointly describe the geometry and physics of each event. They are more informative than using only positions or only energy. The target is not per-event; the target is the source coordinate shared by all events in a source group.

4. Why Split by Source Instead of by Row

The code builds a source_id using Source_X, Source_Y, and Source_Z, and then splits by source. This is a very important design choice.

Why it matters:

  • If rows were randomly split, events from the same source could appear in both train and test sets.
  • That would create data leakage and make performance look better than it really is.
  • By splitting by source, the model must generalize to unseen sources rather than memorize source-specific patterns.

5. Why Use a Heatmap Target Instead of Direct Coordinate Regression

The true source coordinate is converted into a Gaussian heatmap on a 2D grid. This means the model predicts a full spatial distribution instead of only one point estimate.

Reasons for this choice:

  • Heatmap prediction is more stable than direct regression when uncertainty or ambiguity exists.
  • A Gaussian target gives the model a smooth learning signal around the true coordinate rather than a single hard point.
  • The output can later be interpreted in multiple ways: expectation E[x,y], MAP, or uncertainty spread.
  • This fits the physical interpretation better because localization often has uncertainty, not just one exact point.
If the professor asks “why not regress x and y directly?”, the answer is: direct regression gives only one point, while the heatmap retains spatial uncertainty structure and provides a richer training signal.

6. Grid Construction and Why It Uses the Training Set Only

The heatmap grid is computed from the training set only, with 5% padding on each axis. This avoids using information from validation or test targets during target construction.

Why this is good practice:

  • It avoids target leakage.
  • It makes evaluation more honest.
  • The 5% padding prevents edge clipping when sources are near the observed minimum or maximum positions.

7. Padding, Truncation, and the Mask

Each source can have a different number of events. Neural networks prefer fixed-size tensors, so each source is padded or truncated to MAX_EVENTS = 3000.

The mask is crucial because:

  • Padded zeros are not real events.
  • The mask tells the model which entries are valid and which are padding.
  • Attention pooling and ISAB use this mask so the model does not learn from fake padded events.

Without a mask, the model could interpret padded zeros as real physical events, which would hurt performance.

8. Layer-by-Layer Architecture Explanation

8.1 Input Layers

events_in = keras.Input(shape=(max_events, feat_dim), name="events") mask_in = keras.Input(shape=(max_events,), dtype="bool", name="mask")

Two inputs are used because the model needs both the event features and the validity mask. This separation is cleaner than embedding validity as another numeric feature.

8.2 Normalization Layer

norm = KerasNormalization(axis=-1, name="normalize")

This layer standardizes the input features. Different features may have different scales, for example coordinates, angles, and energies. Normalization prevents large-scale features from dominating the gradients.

Why chosen:

  • Improves optimization stability.
  • Makes learning rate behavior more predictable.
  • Reduces sensitivity to raw feature units.

It is adapted on the training tensor only, which is correct practice.

8.3 Shared Dense Event Encoder

The event encoder applies the same dense network to every event. This is a standard set-model idea: each element is first transformed individually before set-level interaction is added.

EVENT_HIDDEN = [256, 256, 128]

There are three residual event blocks.

Block Main Components Purpose
Block 1 Dense 256 -> GELU -> Dropout -> Dense 256 -> LayerNorm + residual projection Lift the raw 6-D feature vector into a richer latent space.
Block 2 Dense 256 -> GELU -> Dropout -> Dense 256 -> LayerNorm + residual Refine event representation while keeping width stable.
Block 3 Dense 128 -> GELU -> Dropout -> Dense 128 -> LayerNorm + residual projection Compress the event representation to a more efficient attention width.

Why dense layers are appropriate here:

  • Each event is represented as a fixed feature vector, so a shared MLP is a natural first-stage encoder.
  • It is simpler and cheaper than attention at the earliest stage.
  • It lets the model learn nonlinear combinations of coordinates, angle, and energy before cross-event reasoning.

8.4 Why Residual Connections Are Used

Residual connections help preserve information and make deeper models easier to optimize. Instead of forcing every block to learn a complete new representation, the block only needs to learn a useful correction to the previous representation.

Why chosen over plain stacked layers:

  • Improves gradient flow.
  • Reduces degradation when depth increases.
  • Allows the network to keep useful low-level information while learning higher-level features.

8.5 Why GELU Activation Is Used

The model uses GELU when USE_GELU = True. GELU is smoother than ReLU and is commonly used in Transformer-style models.

Why GELU instead of ReLU:

  • GELU is smoother and less abrupt than hard-threshold ReLU.
  • It often works well in attention-based architectures.
  • It can provide slightly better representational flexibility for continuous-valued scientific data.

ReLU would also be valid, but GELU is a more modern default for Transformer-like networks.

8.6 Why Dropout Is Used

Dropout with DROPOUT = 0.10 is used in multiple places. Its purpose is regularization.

Why chosen:

  • Reduces overfitting.
  • Encourages more robust hidden representations.
  • Later supports MC Dropout for uncertainty estimation during inference.

This is one of the more important “why used here?” answers: the same mechanism helps both regularization and approximate uncertainty estimation.

8.7 Why Layer Normalization Is Used

Layer normalization is applied inside residual blocks and the attention block. It stabilizes activations and is especially common in attention-based architectures.

Why LayerNorm instead of BatchNorm inside the set/attention part:

  • LayerNorm is less dependent on batch statistics.
  • It is usually more natural for Transformers and variable-structure inputs.
  • BatchNorm can be less ideal when sequence/set masking and variable event counts are involved.

8.8 ISAB: Induced Set Attention Block

This is one of the core architectural choices. The model does not use full self-attention over all events directly. Instead it uses an Induced Set Attention Block (ISAB).

x = ISAB(d_model=units, num_heads=4, m=128, dropout=0.10)(x, mask_in)

How ISAB works conceptually:

  1. Learn a set of inducing points.
  2. Let the inducing points attend to the event set.
  3. Let the event set attend back to the induced summary.

Why ISAB was chosen instead of full self-attention:

  • Full self-attention scales poorly with long sets because it is quadratic in the number of events.
  • Here MAX_EVENTS = 3000, which makes full self-attention expensive.
  • ISAB reduces complexity by routing interaction through m = 128 inducing points.
  • It preserves set-level interaction while remaining computationally practical.

This is one of the strongest justifications in the architecture. If asked “why ISAB?”, the short answer is: to model event-to-event relationships efficiently without the full cost of quadratic self-attention.

8.9 Why Multi-Head Attention Is Used

Multi-head attention allows the model to attend to different interaction patterns in parallel. Different heads can focus on different relational structures among events.

Why 4 heads:

  • Provides multi-view relational modeling.
  • Keeps the attention block expressive without being too expensive.
  • Matches the compressed latent width of 128 reasonably well.

8.10 Feed-Forward Networks Inside ISAB

After each attention step, a feed-forward network is applied. This follows the standard Transformer block pattern: attention mixes information across elements, and the feed-forward network adds nonlinear feature transformation.

The code uses width expansion to 4 * d_model, which is also standard Transformer practice.

8.11 Why Attention Pooling Is Used

After ISAB, the model still has one embedding per event. These must be aggregated into one source-level representation. Instead of simple average pooling, the model uses attention pooling.

Why attention pooling instead of mean pooling:

  • Not all events are equally informative.
  • Attention pooling allows the model to assign higher weight to more useful events.
  • Masking ensures padded events receive no weight.

Mean pooling is simpler and permutation-invariant, but attention pooling is more selective and usually more expressive.

8.12 Why Event Count Is Added as an Extra Feature

The model explicitly appends the normalized number of valid events:

count_feat = sum(mask) / MAX_EVENTS

Why this helps:

  • The number of observed events may itself carry information about confidence or source characteristics.
  • Two sources with similar pooled embeddings but very different event counts may deserve different predictions.
  • Adding this as an explicit feature is cheap and interpretable.

8.13 Prediction Head

The pooled representation plus event count is processed by two dense layers before the final heatmap output.

Purpose of this head:

  • Convert the source-level latent embedding into a spatial probability map.
  • Provide final nonlinear transformation capacity after set aggregation.
  • Separate the source summarization stage from the output distribution stage.

8.14 Final Softmax Heatmap Layer

outputs = layers.Dense(HEATMAP_H * HEATMAP_W, activation="softmax", name="heatmap")(z)

Softmax ensures the output is a valid probability distribution over the 2500 cells.

Why softmax instead of linear output:

  • The target heatmap sums to 1.
  • Softmax enforces nonnegative probabilities that sum to 1.
  • That makes KL divergence and cross-entropy style comparisons meaningful.

9. Loss Function: What It Is and Why It Was Chosen

The training loss is a KL-divergence style loss between the target heatmap and the predicted heatmap.

kl_loss(y_true, y_pred) = mean(sum(y_true * (log(y_true) - log(y_pred))))

Why this is a good choice:

  • The model output is a probability distribution, not just a point estimate.
  • KL divergence is designed to compare probability distributions.
  • It penalizes probability mass placed far from the target distribution.
  • It encourages the predicted heatmap to match the full shape of the target Gaussian heatmap.

9.1 Why Not Mean Squared Error for the Main Loss

Mean Squared Error (MSE) is a common loss for coordinate regression, but it is not the best main loss for this specific setup.

Why MSE was not chosen as the main loss here:

  • The target is a distribution over 2500 cells, not just two scalar coordinates.
  • MSE treats all output dimensions independently and does not naturally respect the “sum to one” probability structure.
  • KL divergence is more meaningful when both target and prediction are probability distributions.
If asked “what is MSE and why not use it?”, answer: MSE measures squared numeric error, which is excellent for direct scalar regression, but this model predicts a probability distribution, so KL divergence is a more natural match than MSE.

9.2 Why Categorical Cross-Entropy Appears in the Logs

The code compiles the model with metrics=["categorical_crossentropy"]. This is used as a diagnostic metric, not as the optimized loss. It provides another distribution-matching view that is easy to monitor during training.

10. Optimizer and Training Choices

10.1 Adam Optimizer

keras.optimizers.Adam(LEARNING_RATE, clipnorm=1.0)

Why Adam is used:

  • It adapts learning rates per parameter.
  • It is strong for deep nonlinear models and attention-based networks.
  • It is a practical default when feature scales and gradient patterns vary.

10.2 Why Gradient Clipping Is Used

clipnorm=1.0 is used to control large gradients. This can improve stability, especially with attention layers and long inputs.

10.3 Early Stopping

Early stopping is configured with patience=7. This prevents unnecessary training after validation loss stops improving.

Why chosen:

  • Reduces overfitting.
  • Saves training time.
  • Restores the best weights automatically.

10.4 ReduceLROnPlateau

This callback lowers the learning rate if validation loss stalls. The purpose is to allow finer convergence after the initial faster learning phase.

10.5 ModelCheckpoint

The best validation model is saved to best.weights.h5. This ensures that evaluation uses the strongest observed model, not just the final epoch.

10.6 Stochastic Weight Averaging

The code tries to add Stochastic Weight Averaging when available. SWA can improve generalization by averaging multiple solutions found during training.

11. Why MC Dropout Is Used at Inference

The code performs MC_DROPOUT_PASSES = 10 stochastic forward passes during evaluation.

Purpose:

  • Approximate predictive uncertainty without changing the main model design.
  • Average multiple stochastic predictions to get a smoother and often more robust heatmap.
  • Exploit the fact that dropout can be interpreted as approximate Bayesian inference at test time.

This is a useful answer if the professor asks why dropout was kept active during inference.

12. How Final Coordinates Are Extracted from the Heatmap

12.1 Expected Coordinate E[x, y]

One evaluation mode computes the expected x and y values from the full probability map. This uses all probability mass, not just the single most likely cell.

Why useful:

  • Smoother estimate.
  • Uses the full distribution.
  • Often less noisy than hard argmax.

12.2 MAP Estimate

The Maximum A Posteriori (MAP) estimate uses the highest-probability grid cell. This gives the single most likely discrete location.

12.3 Subpixel Refinement

The code refines the MAP estimate using a local parabolic fit around the peak. This helps overcome grid discretization.

Why useful:

  • The heatmap grid is discrete, but the true source coordinate is continuous.
  • Subpixel refinement improves precision without increasing the grid size.

13. Metrics and Why They Matter

13.1 MAE_x and MAE_y

Mean Absolute Error in x and y measures the average absolute localization error along each axis.

Why useful:

  • Easy to interpret in physical units.
  • Shows whether one axis is systematically harder than the other.

13.2 Euclidean MAE

This is the average straight-line localization error between predicted and true source coordinates. It is usually the most intuitive single-number localization metric.

13.3 R² in x, y, and Joint XY

R² measures how well predictions explain the variance in the true coordinates.

Why R² is useful here:

  • It gives a regression-quality interpretation in addition to absolute error.
  • It shows whether predictions follow the trend of the true source positions well.
  • The joint XY version evaluates both coordinates together.

13.4 Mean Squared Error: What It Means Even Though It Is Not Used Here

Mean Squared Error (MSE) is the average of squared differences between prediction and target. It penalizes large errors more strongly than MAE.

Why professors ask about it:

  • It is one of the most common regression losses and metrics.
  • It is mathematically convenient and differentiable.
  • It is often used when predicting continuous scalar outputs directly.

Why this model reports MAE and R² instead:

  • MAE is directly interpretable in source-position units.
  • R² shows explanatory quality.
  • The training target is a heatmap distribution, so KL-based training is more natural than MSE-based training.

14. Why This Is a Set Model Rather Than an RNN or Plain CNN

The input is fundamentally a set of events per source. Event order is not the physical signal of interest.

Why not an RNN:

  • RNNs assume ordered sequential dependence.
  • That assumption is weak or artificial here.

Why not a plain CNN over the event list:

  • A CNN would impose local neighborhood assumptions along event index order.
  • Those neighborhoods are not physically meaningful unless event order itself has meaning.

Why a set architecture fits better:

  • It is permutation-aware or permutation-invariant where needed.
  • It allows event-level encoding plus cross-event interaction plus global pooling.
  • It naturally handles variable event counts using masking.

15. Important Regularization and Stability Choices

  • L2 = 1e-6: mild weight decay to discourage unnecessarily large weights.
  • DROPOUT = 0.10: regularization plus MC Dropout support.
  • LayerNorm: stabilizes attention and residual blocks.
  • Gradient clipping: prevents unstable updates.
  • Early stopping: prevents overtraining after validation performance plateaus.

16. Important Practical Outputs of the Model

  • best.weights.h5: best validation model.

17. Short “Why This, Not That?” Answers for Oral Questions

Professor’s likely question Short answer
Why use a heatmap instead of direct x-y regression? A heatmap captures uncertainty and gives a richer spatial learning signal than a single point estimate.
Why KL divergence instead of MSE? The output is a probability distribution over a grid, so KL divergence is more appropriate than scalar regression loss.
Why ISAB? It models interactions among many events more efficiently than full self-attention.
Why attention pooling instead of average pooling? Because some events are more informative than others, and attention learns how much each event should contribute.
Why GELU? It is a smooth modern activation that often works well in Transformer-like models.
Why LayerNorm instead of BatchNorm in the attention stack? LayerNorm is better suited to Transformer-style architectures and less dependent on batch statistics.
Why dropout? It regularizes training and also enables MC Dropout uncertainty estimation at inference time.
Why split by source? To avoid leakage and ensure evaluation is on unseen sources, not unseen rows from the same source.

18. Final Summary

This model is a set-based attention architecture for source localization. It converts each source’s event set into a learned latent representation, models relationships among events using ISAB, aggregates the set using attention pooling, and predicts a 2D probability heatmap rather than only a point coordinate. The use of a heatmap target, KL-divergence loss, attention-based aggregation, masking, and MC Dropout all serve a common goal: stable and physically meaningful source localization from variable-sized event sets.
